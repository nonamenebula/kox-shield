## Шаг 1: Установить Entware

> Пропустите если Entware уже установлен: `ls /opt/bin/opkg` — должен существовать.

### 1.1 Включить компонент OPKG

1. Откройте веб-интерфейс роутера: **http://192.168.1.1**
2. Перейдите в **Управление → Компоненты**
3. Найдите **«Поддержка пакетов OPKG»** → нажмите **Установить**
4. Дождитесь перезагрузки

### 1.2 Подготовить USB-флешку

Отформатируйте флешку в **EXT4** или **NTFS** (≥ 1 ГБ). На macOS используйте Homebrew:

```bash
brew install e2fsprogs
sudo /usr/local/opt/e2fsprogs/sbin/mkfs.ext4 /dev/diskN   # замените N на номер диска
```

### 1.3 Установить Entware

1. Вставьте флешку в USB роутера
2. Откройте веб-интерфейс → **Приложения → Entware**
3. Нажмите **«Установить»** напротив нужного раздела

### 1.4 Проверить

```bash
ssh root@192.168.1.1 -p 222
# пароль: keenetic (или ваш, если меняли)
ls /opt/bin/opkg    # должен существовать
```

## Шаг 2: Получить VLESS-подписку

### Вариант А: Подписка KOX Shield (рекомендуется)

Зарегистрируйтесь на **[kox.nonamenebula.ru/register](https://kox.nonamenebula.ru/register)**:
- Получите URL подписки вида `https://kox.nonamenebula.ru/c/TOKEN`
- Несколько серверов на выбор (Германия, Финляндия, США, и др.)
- Автообновление параметров

### Вариант Б: Свой VLESS-сервер

Поднимите Xray на VPS (Ubuntu/Debian):

```bash
# Установка Xray
bash <(curl -L https://github.com/XTLS/Xray-install/raw/main/install-release.sh)

# Генерация ключей
xray x25519                       # privateKey + publicKey
openssl rand -hex 4               # shortId
cat /proc/sys/kernel/random/uuid  # UUID
```

Ваша VLESS-ссылка:
```
vless://UUID@YOUR-IP:443?security=reality&sni=www.microsoft.com&fp=chrome&pbk=PUBLIC-KEY&sid=SHORT-ID&flow=xtls-rprx-vision#MyServer
```

## Шаг 3: Установить KOX Shield

Подключитесь по SSH и выполните **одну команду**:

```bash
ssh root@192.168.1.1 -p 222

curl -fsSL https://raw.githubusercontent.com/nonamenebula/kox-shield/main/install.sh | sh
```

Установщик в интерактивном режиме:

1. Проверит интернет-соединение
2. Предложит удалить Kvass / Shadowsocks / sing-box (если найдёт)
3. Установит пакеты: `xray-core`, `curl`, `jq`, `cron`, `iptables`
4. Спросит URL подписки или VLESS-ссылку
5. При нескольких серверах в подписке — предложит выбрать
6. Сгенерирует `config.json` с вашими доменами
7. Настроит iptables (только порты 80/443)
8. Установит watchdog-мониторинг
9. Запустит Xray и проверит соединение

## Шаг 4: Настроить Telegram Bot (необязательно)

Бот позволяет управлять VPN прямо из Telegram — включать/выключать, переключать серверы, смотреть статус.

### 4.1 Создать бота

1. Напишите [@BotFather](https://t.me/BotFather) в Telegram
2. Отправьте `/newbot` и следуйте инструкциям
3. Скопируйте **токен** вида `1234567890:AAF...`

### 4.2 Узнать свой Telegram ID

Напишите [@userinfobot](https://t.me/userinfobot) — он пришлёт ваш ID.

### 4.3 Прописать в конфиге роутера

```bash
ssh root@192.168.1.1 -p 222

# Задать токен и ID
kox admin set 123456789          # ваш Telegram ID

# Добавить токен бота
vi /opt/etc/xray/kox.conf
```

В файле `/opt/etc/xray/kox.conf` добавьте или измените:
```
KOX_BOT_TOKEN="1234567890:AAF..."
KOX_ADMIN_ID="123456789"
KOX_SUB_URL="https://kox.nonamenebula.ru/c/YOUR_TOKEN"
```

### 4.4 Запустить бота

```bash
/opt/etc/init.d/S90kox-bot start
```

Напишите боту `/start` — должно появиться меню управления.

## Шаг 5: Проверить работу

```bash
# Статус (всё должно быть ✅)
kox status

# Вывод:
# Xray:         ✅ запущен
# Порт 10808:   ✅
# iptables:     ✅
# VPN туннель:  ✅ ВКЛЮЧЕН
# Сервер:       147.78.181.52
```

**Проверка из браузера:**
1. Откройте **2ip.ru** на устройстве подключённом к роутеру
2. IP должен отличаться от вашего домашнего — это значит трафик идёт через VPN
3. Откройте YouTube, Instagram — должны работать
4. Откройте Госуслуги, банк — должны работать с вашим домашним IP

## Управление через консоль `kox`

```bash
# ── Статус и управление ────────────────────────────────────────────
kox status              # состояние Xray, iptables, VPN, сервер
kox on                  # включить VPN (применить iptables)
kox off                 # выключить VPN (интернет напрямую, Xray работает)
kox restart             # перезапустить Xray
kox test                # проверить корректность config.json
kox server              # параметры текущего VLESS-сервера
kox stats               # статистика трафика iptables

# ── Домены ────────────────────────────────────────────────────────
kox add example.com     # добавить домен в туннель, Xray перезапустится
kox del example.com     # удалить домен из туннеля
kox check example.com   # через какой канал идёт домен (VPN или прямой)?
kox list                # список всех доменов в туннеле

# ── IP и подсети ──────────────────────────────────────────────────
kox add-ip 1.2.3.0/24   # добавить IP/подсеть в туннель
kox del-ip 1.2.3.0/24   # удалить IP/подсеть из туннеля
kox list-ip             # список всех IP/подсетей в туннеле

# ── Категории доменов (готовые списки) ────────────────────────────
kox list-cats                  # посмотреть все доступные категории
kox list-load telegram         # загрузить категорию по slug
kox list-load 1 3 5            # загрузить несколько по номерам
kox list-load all              # загрузить все категории сразу
kox list-remove telegram       # удалить категорию из туннеля
kox list-remove all            # удалить все категории
kox list-check                 # проверить наличие обновлений списков
kox list-update                # обновить списки с GitHub и применить

# ── Серверы и подписка ────────────────────────────────────────────
kox sub set <URL>       # задать URL подписки (сохраняется в kox.conf)
kox sub get             # показать текущий URL подписки
kox update-sub          # обновить параметры сервера из подписки + перезапуск Xray
kox cron-on             # включить авто-обновление из подписки (ежедневно 04:00)
kox cron-off            # отключить авто-обновление

# ── Диагностика и логи ────────────────────────────────────────────
kox log                 # последние 30 строк лога ошибок Xray
kox log-live            # логи Xray в реальном времени (Ctrl+C для выхода)
kox clear-log           # очистить xray-err.log, xray-acc.log, kox-bot.log
kox watchdog-log        # лог watchdog (авто-восстановление при сбое)

# ── Резервные копии ───────────────────────────────────────────────
kox backup              # создать бэкап config.json с временной меткой
kox restore             # интерактивный выбор бэкапа для восстановления
kox restore config_20260501_120000.json   # восстановить конкретный файл

# ── Telegram Bot ──────────────────────────────────────────────────
kox bot                 # статус бота (запущен/остановлен, PID)
kox bot-setup           # мастер первичной настройки (токен + admin ID)
kox admin set 123456789 # назначить Telegram-администратора по ID
kox admin show          # показать текущего администратора

# ── Обновление ────────────────────────────────────────────────────
kox upgrade             # проверить и установить новую версию KOX Shield
kox upgrade --force     # обновить без подтверждения (используется ботом)

# ── Очистка устаревшего ПО ────────────────────────────────────────
kox clean-legacy        # обнаружить Kvass, Shadowsocks, SOCKS-интерфейсы
kox clean-legacy --force  # удалить без подтверждения

# ── Справка ───────────────────────────────────────────────────────
kox help                # полный список команд с описанием
```

## Переключение серверов

### Через Telegram бот (удобнее)

1. Нажмите **🌐 Серверы** в главном меню
2. Нажмите **🔀 Сменить сервер из подписки**
3. Бот покажет список серверов с именами и пингом:
   ```
   🟢 🇩🇪 Германия | Klever  🟢 39 ms
   🟢 🇩🇪 Германия           🟢 36 ms
   🟡 🇺🇸 США                🟡 111 ms
   🟢 🇨🇿 Чехия              🟢 38 ms
   ✅ 🟢 🇫🇮 Хельсинки       🟢 28 ms  ← текущий
   ```
4. Нажмите на нужный сервер
5. При сбое — конфиг автоматически вернётся к предыдущему серверу

### Через консоль

```bash
# Задать URL подписки (если не задан)
kox sub set https://kox.nonamenebula.ru/c/TOKEN

# Обновить из подписки (берёт первый сервер)
kox update-sub
```

## Устранение проблем

### Интернет не работает после включения VPN

KOX Shield перехватывает только HTTP/HTTPS (порты 80/443). Если проблема:

```bash
kox off              # выключить VPN — интернет должен сразу заработать
kox log              # посмотреть ошибки Xray
kox test             # проверить конфиг
kox status           # проверить состояние
```

### Xray не запускается

```bash
kox log              # посмотреть лог ошибок
tail -30 /opt/var/log/xray-err.log

# Проверить конфиг
kox test

# Перезапустить
kox restart
```

### После перезагрузки роутера VPN не включается

```bash
# Проверить автозапуск
ls /opt/etc/init.d/S24xray    # Xray
ls /opt/etc/init.d/S90kox-bot # Бот

# Применить iptables вручную
sh /opt/etc/ndm/netfilter.d/99-kox-nat.sh

# Запустить Xray
/opt/etc/init.d/S24xray start
```

### Бот не отвечает / пропало меню

```bash
# Перезапустить бота
/opt/etc/init.d/S90kox-bot restart
tail -10 /opt/var/log/kox-bot.log
```

В Telegram напишите боту `/start` — меню появится как новое сообщение.

### Трафик не идёт через VPN

```bash
# Проверить iptables
iptables -t nat -L XRAY_REDIRECT -n -v
# Должны быть REDIRECT правила на порты 80/443

# Восстановить если пусто
sh /opt/etc/ndm/netfilter.d/99-kox-nat.sh

# Проверить что домен в списке
kox check youtube.com
```

### Конфликт с Kvass

```bash
# Остановить Kvass
/opt/etc/init.d/S96kvas stop 2>/dev/null
opkg remove kvas 2>/dev/null

# Или запустить автоматическую очистку через бота
# Кнопка: Инструменты → Очистить устаревшее ПО
```
